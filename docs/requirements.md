## 1. プロジェクト概要

### 1.1 目的
linkle株式会社の企業認知度向上を目的としたコーポレートサイトの構築

### 1.2 ターゲットユーザー

## 3. サイト構成

### 3.1 必要ページ一覧

#### 1. トップページ (`/`)
- メインビジュアル
- サービス概要セクション
- 会社紹介セクション
- 最新ニュース表示（3件程度）
- お問い合わせへの導線

#### 2. 会社概要 (`/about`)
- 会社情報（上記2.会社情報の内容）
- 会社所在地（Google Maps埋め込み）
- アクセス情報

#### 3. サービス紹介 (`/service`)
- Web制作サービスの詳細
- 得意分野の紹介
- 技術スタック
- 開発フロー
- お問い合わせへの導線

#### 4. ニュース・お知らせ (`/news`)
- ニュース一覧ページ
- ニュース詳細ページ
- 日付順ソート機能
- ページネーション

#### 5. 採用情報 (`/recruit`)
- 募集職種：フルスタックエンジニア
- 求める人材像：技術が大好きな方
- 福利厚生（セクションのみ用意・内容は後日記入）
- 選考フロー（セクションのみ用意・内容は後日記入）
- 応募方法（セクションのみ用意・内容は後日記入）
- 応募フォームへの導線

#### 6. お問い合わせ (`/contact`)
- お問い合わせフォーム
- 送信完了ページ

### 3.2 共通要素
- ヘッダーナビゲーション
  - ロゴ
  - グローバルナビゲーション（Home / About / Service / News / Recruit / Contact）
  - レスポンシブ対応（ハンバーガーメニュー）

- フッター
  - 会社情報（簡易版）
  - ナビゲーションリンク
  - プライバシーポリシーリンク
  - コピーライト表記

## 4. 機能要件

### 4.1 お問い合わせフォーム

**フォーム項目：**
- お問い合わせ種別（選択式）
  - サービスについて
  - 採用について
  - 取材について
  - その他
- 氏名（必須）
- メールアドレス（必須・バリデーション付き）
- 会社名（任意）
- お問い合わせ内容（必須・テキストエリア）

**送信先：** one@linkle.group

**バリデーション：**
- 必須項目チェック
- メールアドレス形式チェック
- 送信前の確認画面または確認モーダル

**送信後：**
- サンクスページへリダイレクト
- 自動返信メール送信（オプション）

### 4.2 ニュース・お知らせ機能

**更新方法：**
- 開発者が直接コードで更新
- JSONファイルまたはMarkdownファイルでコンテンツ管理
- カテゴリ分け不要

**表示機能：**
- 一覧表示（日付降順）
- 詳細ページ
- トップページに最新3件表示
- ページネーション（10件/ページ）

---

## 5. デザイン要件

### 5.1 デザインコンセプト
- モダンでクリーンなデザイン
- 信頼感と技術力を感じさせるビジュアル
- 参考サイト：https://neu-ad.jp/

### 5.2 カラースキーム
- **ベースカラー**: #6ccaf1（青系）
- アクセントカラー：ベースカラーを基調とした配色
- テキストカラー：#333333（濃いグレー）
- 背景色：#FFFFFF（白）、#F8F9FA（薄いグレー）

### 5.3 タイポグラフィ
- 日本語：游ゴシック体、Noto Sans JP、またはモダンなゴシック体
- 英語：Sans-serif系のモダンなフォント
- 見出し：太字で視認性の高いサイズ
- 本文：読みやすい行間・文字サイズ

### 5.4 レイアウト
- レスポンシブデザイン必須
  - デスクトップ（1200px以上）
  - タブレット（768px〜1199px）
  - スマートフォン（767px以下）
- 余白を活かした読みやすいレイアウト
- スクロールアニメーション（フェードイン等）

---

## 6. 技術要件

### 6.1 フロントエンド
- **フレームワーク**: React 19.2.0
- **ビルドツール**: Create React App
- **ルーティング**: React Router v6
- **スタイリング**:
  - CSS Modules / Styled Components / Tailwind CSS（選択）
  - レスポンシブ対応必須
- **アニメーション**: Framer Motion / React Spring（選択）

### 6.2 フォーム処理
- **バリデーション**: React Hook Form + Yup/Zod
- **メール送信**:
  - EmailJS / Formspree / 独自バックエンドAPI（選択）
  - 送信先：one@linkle.group

### 6.3 ニュース管理
- **データ管理**:
  - JSONファイル（`/public/data/news.json`）
  - または Markdownファイル + gray-matter
- **更新方法**: Git経由で直接コミット

### 6.4 パフォーマンス要件
- ページ読み込み時間：3秒以内
- 画像の最適化（WebP形式推奨）
- Lazy Loading実装
- SEO対策（メタタグ、OGP設定）

### 6.5 ブラウザ対応
- Chrome（最新版）
- Firefox（最新版）
- Safari（最新版）
- Edge（最新版）
- モバイルブラウザ対応

---

## 7. SEO・アクセシビリティ要件

### 7.1 SEO
- 各ページに適切なタイトルタグ・メタディスクリプション
- OGP（Open Graph Protocol）設定
- 構造化データ（企業情報）
- sitemap.xml生成
- robots.txt設定

### 7.2 アクセシビリティ
- セマンティックHTML使用
- 適切な見出しレベル（h1〜h6）
- altテキスト設定
- キーボード操作対応
- カラーコントラスト確保

---

## 8. 多言語対応

- **対応言語**: 日本語のみ
- 将来的な多言語対応の拡張性は考慮しない

---

## 9. セキュリティ要件

- フォーム送信時のCSRF対策
- XSS対策
- メールアドレスの難読化
- HTTPS通信必須（デプロイ時）
